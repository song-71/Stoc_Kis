# Stoc_Kis 프로젝트 지침

## 대화 규칙
- **사용자와 대화할 때는 항상 존칭(높임말)을 사용한다.** 반말·평어체 금지. 모든 응답은 "~합니다 / ~하겠습니다 / ~입니다" 형태의 존댓말로 작성한다.
- **가능한 한 신조어·외래어·영어 약어를 쓰지 않고 이해하기 쉬운 우리말로 설명한다.** 예: "밴/블록"→"차단", "스로틀(throttle)"→"접속 제한/조이기", "백오프(backoff)"→"재시도 대기시간", "off-hours"→"장 시간 외". 우리말로 충분히 전달되면 외래어를 쓰지 않는다. 기술용어(WSS·핸드셰이크 등)가 불가피하면 **처음 쓸 때 한 번 쉬운 말로 풀어** 준다("핸드셰이크=서버와 연결을 맺는 첫 단계").
- **현재 시각은 절대 사용자에게 묻지 않는다. 항상 `TZ=Asia/Seoul date` 로 스스로 확인해 판단한다.** 서버는 UTC 이므로 직접 KST 를 구할 것. "지금 몇 시인지 / 장중인지" 를 사용자에게 확인받으려 하지 말고, 필요하면 조용히 date 로 확인한 뒤 진행한다.
- 시각·장운영시간(개장 09:00·마감 15:30·시간외 등)이 조금이라도 관련된 판단(프로세스 기동 여부, 로그 해석, "지금/방금/오늘" 등)에서는 과거 맥락의 시각으로 말하지 말고 **그 시점의 실제 KST 기준**으로 판단·서술한다. (시각이 무관한 작업에서는 굳이 시각을 앞세우지 않는다.)

## 문제 분석·디버깅 규칙 (필수)
- **이상신호에는 "이게 정상인가?"가 아니라 "원인이 반드시 있다"로 출발한다.** 사용자가 이상을 제기하면 버퍼·수신율 같은 표면 수치로 "정상"이라 합리화하지 말고, 코드와 로그에서 메커니즘을 끝까지 추적해 근본원인을 특정한다. "어제도 비슷했다" 식의 어림 비교로 넘기지 않는다.
- **코드·로그 확인은 사용자 확인 없이 자율로 즉시 수행한다.** "~를 확인해볼까요? / 확인하겠습니다(허락 대기)" 식으로 묻지 말고, 파일 읽기·grep·로그 분석을 바로 진행한 뒤 **결과만 보고**한다. (읽기 전용 진단은 `rules/trading_project_main_plan.md` 불변원칙 7 및 `.claude/settings.local.json` 의 Read 허용으로 보장됨. 수정·삭제·이동은 별도 확인 필요.)
- **단정에는 증거(`파일:라인` / 로그 원문)를 반드시 동반한다.** 확정과 가설을 구분해 표기하고("확정"/"유력 가설"/"미확정"), 추측은 추측이라 명시한다.
- **"제가 틀렸습니다"로 얼버무리고 넘어가지 않는다.** 오판을 인정하면 반드시 그 자리에서 진짜 원인을 끝까지 규명한다. 검증되지 않은 분석 위에 코드를 고치지 않는다.
