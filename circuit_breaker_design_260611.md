# 서킷브레이커 대응 동작 설계안 (260611)

_근거: KRX 규정 (찾기쉬운 생활법령정보 + 한국거래소 제도). 구현 전 검토용 초안._

---

## 0. 목적
1. **서킷브레이커(CB) 발동 시 30분간 실시간 체결 무수신을 "장애"로 오인해 접속을 두드리지 않게 한다** → 접속 폭주 → KIS 의 WSS 접속 IP 차단 → IP 교체 반복, 이 악순환을 막는 것이 1순위.
2. CB 단계별 시장 동작에 맞춰 **예상체결가 수신 ↔ 실시간 체결가** 를 제때 전환한다.

## 1. 시장 동작 요약 (근거 확정)
- **1·2단계**: 20분 전면 매매중단(취소호가만 가능) → 이후 10분 단일가(동시호가)로 호가접수, 이 구간에 **예상체결가 공표** → 합치가격으로 재개. (총 30분)
- **3단계**: **당일 장 종료**.
- 각 단계 **1일 1회**, **14:50 이후 1·2단계 발동 안 함**.
- **사이드카는 대상 아님**(프로그램매매 호가효력만 5분 정지, 전면중단 아님 → 일반매매 계속).

### 봇 관점 데이터 흐름 (1·2단계)
```
발동 ──[20분]── 실시간 체결 0 ──[10분]── 예상체결가 수신 ──> 재개: 실시간 체결
```

## 2. 감지 (장운영정보 H0STMKO0)
- 시장별 상시구독 종목(삼성전자=KOSPI / 카카오게임즈=KOSDAQ)으로 장운영정보를 항상 수신.
- 발동/해제를 `mkop_cls_code` **또는** `antc_mkop_cls_code` 코드값으로 감지(현재값이 비고 예상값으로만 오는 경우 대비 — 260611 적용 완료).
- **★ 정확한 코드값은 다음 실발동 때 `[장운영정보]` 로그로 실측 확정 후 매핑.**
  - 현재 후보: `174`=서킷발동 / `184`=서킷개시 / `164`=시장임시정지 / `175`·`185`=해제. (단 실측에서 다른 값/필드로 올 가능성 있어 **확정 전 자동전환 활성화 금지**)
- 사이드카(`187`/`388`/`397`/`398`)는 정보성 로깅만, CB 동작에서 제외.

## 3. 발동 시 동작 — 1·2단계
1. **상태 기록**: `_market_wide_event[market]` 세트, 발동시각 기록, **자동복귀 예정시각 `_cb_resume_at[market] = 발동시각 + 30분`** 설정.
2. **무수신 감시 정지** ★최우선: 해당 CB 활성 동안 무수신 감시(재구독·강제재접속)를 억제. (no-data 억제 조건에 `_market_wide_event` 비어있지 않으면 포함)
3. **WSS 연결 유지**: 절대 끊거나 재접속하지 않음(서버 ping/pong 으로 유지). → 접속 시도 0 → IP 차단 원천 차단.
4. **해당 시장 전 종목 → 예상체결가(H0STANC0) 전환**: `_desired_subscription_map` 이 CB 시장 종목을 예상체결로 매핑 + 즉시 재적용. (20분 중단 구간엔 예상체결도 거의 없지만 무해, 10분 단일가 구간에 수신)
5. **신규 매매 보류**(사용자 확인 필요): CB 활성 동안 해당 시장 신규 매수 진입 금지. 보유 매도 판단도 재개까지 보류(취소만 가능한 시장 상태와 정합). — *트레이딩 정책이라 확정 필요.*

## 4. 재개 시 동작
- **재개 신호(해제 코드) 수신 또는 `_cb_resume_at` 도달, 둘 중 먼저** 발생 시:
  1. `_market_wide_event[market]` 클리어
  2. 무수신 감시 재개
  3. 해당 시장 전 종목 → **실시간 체결가 복귀** + 즉시 재적용
  4. 신규 매매 판단 재개
- **안전장치**: 코드 신호를 놓쳐도 +30분 타이머로 자동 복귀(무한 정지 방지). 반대로 +30분 전 해제코드 오면 즉시 복귀.

## 5. 3단계 (당일 장종료)
- 3단계 감지 시 그날 정규장 마감과 동일 처리: 전 종목 구독 정리/종료, 신규매매 종료. (기존 15:30 마감 로직 재사용)
- 데이터 저장·마무리는 정상 수행.

## 6. 시간·충돌 가드
- **14:50 이후** 1·2단계 발동 불가 → 그 이후 CB류 코드가 오면 **의심 로깅**(오탐/장마감 단일가 혼동 방지).
- **장 마감 동시호가(15:20~15:30)** 의 단일가 코드와 CB 코드가 겹치지 않는지 확인 — 기존 마감 단일가 전환 로직과 **충돌 금지**.

## 7. 적용 우선순위 (안전 순서)
1. **(이미 적용) `[장운영정보]` 로그 + 감지필드 보강** → 코드값 실측 준비.
2. **무수신 감시 정지(§3-2) + WSS 유지(§3-3)** — 이것만 먼저 넣어도 "CB 시 IP 재차단" 은 막힘. **가장 낮은 위험, 가장 큰 효과.**
3. **예상체결 전환(§3-4) + 재개 복귀(§4)** — 코드값 실측 확정 후.
4. **3단계 종료(§5), 매매 보류(§3-5)** — 정책 확정 후.

## 8. 미확정 (실측·확인 필요)
- 정확한 `mkop`/`antc_mkop` 코드값 (다음 실발동 로그).
- 단일가 구간 예상체결 실제 수신 여부 (실측).
- KOSPI/KOSDAQ 동시 발동 시 각각 독립 처리 — 코드별 market 매핑으로 자연 처리되나 실측 확인.
- §3-5 신규매매 보류 정책 — 사용자 결정.
