# 상한가 근접 매수 전략 (25~28% 구간 진입)

> 작성일: 2026-04-20
> 목적: 25~28% 상승 종목 중 상한가(30%) 도달 가능성 높은 종목을 선별 매수
> 관련 문서: `docs/KIS New_str_api_260402.md` (보조 API 상세)

---

## 1. 전략 개요

### 컨셉
```
전일종가 대비 25~28% 상승 중인 종목 감지
  → 상한가(30%) 도달 가능성 판단 (다중 필터)
  → 높은 확률 종목만 매수
  → 상한가 도달 시 익일 전략 연계 / 미도달 시 손절
```

### 기존 전략과의 관계

| 전략 | 진입 시점 | 목표 |
|------|----------|------|
| **VI 매수** (기존) | VI 발동 중 (등락률 무관) | VI 해제 후 반등 차익 |
| **종가 매수** (기존) | 15:20 동시호가 | 익일 상승 기대 |
| **상한가 근접 매수** (신규) | 장중 25~28% 도달 시 | 상한가(30%) 도달 차익 |

### 핵심 차이: VI 매수와 다른 점
- VI 매수: VI 발동 → 예상체결가 비교 → 해제 후 반등 노림
- 상한가 근접 매수: **VI 무관하게** 25~28% 구간 도달 시 상한가 도달 가능성 판단 → 매수
- 다만 VI 발동 중인 종목도 조건 충족 시 매수 가능 (중복 아닌 보완)

## 2. 감지 메커니즘

### 2-1. 실시간 감지 (WSS 기반)

현재 WSS ingest_loop에서 이미 `prdy_ctrt`(전일대비등락률)을 실시간 추적 중.

```
[현재 코드 흐름] (ws_realtime_trading.py:7658)
WSS 체결 데이터 수신 → prdy_ctrt 추출 → 29%+ 시 긴급매도 모니터링

[추가할 흐름]
WSS 체결 데이터 수신 → prdy_ctrt 추출
  → 25~28% 구간 진입 감지 → 상한가 근접 매수 판단 로직 실행
```

**장점**: 이미 구독 중인 종목은 추가 API 호출 없이 즉시 감지
**한계**: WSS 구독 중인 종목(최대 40 슬롯)에서만 감지 가능

### 2-2. 주기적 스캔 (REST API 보완)

WSS 구독 외 종목도 포착하기 위해 REST API 병행:

#### capture_uplowprice API (상하한가 포착)
| 항목 | 내용 |
|------|------|
| TR_ID | `FHKST130000C0` |
| URL | `/uapi/domestic-stock/v1/quotations/capture-uplowprice` |
| 핵심 파라미터 | `FID_DIV_CLS_CODE="3"` → **25% 상한가 근접 종목** |

```
FID_PRC_CLS_CODE = "0"    # 상한가
FID_DIV_CLS_CODE = "3"    # 25% 근접 (= 25~30% 구간)
                           # "2": 20% 근접, "1": 15% 근접
                           # "6": 8% 근접, "5": 10% 근접
```

#### fluctuation API (등락률 순위)
| 항목 | 내용 |
|------|------|
| TR_ID | `FHPST01700000` |
| URL | `/uapi/domestic-stock/v1/ranking/fluctuation` |
| 핵심 파라미터 | `fid_rsfl_rate1="25"`, `fid_rsfl_rate2="28"` → **25~28% 범위 필터** |

**이미 `fetch_top30_each_1m.py`에서 사용 중** → 파라미터만 추가하면 됨

#### 스캔 주기
```
09:10 ~ 14:50: 매 3분마다 capture_uplowprice + fluctuation 조회
  → 25~28% 종목 리스트 갱신
  → 미구독 종목은 WSS 동적 추가 (슬롯 여유 시)
```

## 3. 매수 판단 필터 (Signal Strength 적용)

### 3-1. 상한가 도달 확률을 좌우하는 요인 (중요도 순)

| 순위 | 요인 | 설명 | 측정 방법 |
|------|------|------|----------|
| 1 | **재료 강도** | 테마/공시/뉴스 기반 상승인지 | 자동 판단 어려움 → 간접 지표 활용 |
| 2 | **거래량 + 체결강도** | 매수세의 지속력 | Volume Power API + WSS 거래량 |
| 3 | **호가 잔량 비율** | 매수벽 vs 매도벽 | WSS 호가 데이터 (H0STASP0) |
| 4 | **외국인/기관 동향** | 스마트머니 참여 여부 | frgnmem_pchs_trend API |
| 5 | **변동성 상태** | 저변동성 돌파 vs 고변동성 잡음 | 10일 변동성 (사전 계산) |
| 6 | **VI 해제 후 반응** | VI 해제 후 재상승 = 가장 강한 시그널 | WSS 실시간 체결가 |

### 3-2. 매수 강도 점수 산출

```
[기본 점수] 0.40 (25~28% 구간 진입)

[가산 항목]
+0.20  체결강도 상위 20% (Volume Power API)
+0.15  호가 매수잔량 > 매도잔량 × 2.0 (WSS 호가)
+0.10  외국인 최근 3일 순매수 (frgnmem_pchs_trend)
+0.10  저변동성 돌파 (10일 vol < 2%)
+0.10  VI 해제 후 1분 내 재상승 중
+0.05  거래량 전일 대비 300% 이상

[감산 항목]
-0.20  호가 매도잔량 > 매수잔량 × 3.0 (매도벽)
-0.15  체결강도 하위 30%
-0.10  고변동성 (10일 vol > 5%)
-0.10  외국인 최근 3일 순매도
-0.05  장 후반 (14:30 이후) — 상한가 도달 시간 부족

[매수 결정]
0.7+   → 매수 (정상 수량)
0.5~0.7 → 소량 매수 (최소 수량)
0.5 미만 → 매수 보류
```

### 3-3. 추가 필수 조건 (점수와 무관하게 통과해야 함)

```
1. 관리종목/투자경고 아닐 것 (iscd_stat_cls_code via KRX DB)
2. 전일종가 ≥ 1,000원 (저가주 제외)
3. 당일 거래량 ≥ 100,000주 (유동성 확보)
4. 이미 보유 중이 아닐 것
5. 가용 현금이 1주 이상 매수 가능할 것
```

## 4. 매수 실행

### 4-1. 주문 방식

| 상황 | 주문 유형 | 가격 |
|------|----------|------|
| **VI 미발동 상태** | 지정가 (`ord_dvsn="00"`) | 현재가 + 1~2호가 |
| **VI 발동 중** | 지정가 (`ord_dvsn="00"`) | 상한가 (`stck_mxpr`) |

- 시장가 주문은 슬리피지 리스크 → 지정가 선호
- VI 발동 중이면 상한가로 주문 (VI 해제 시 체결)
- 상한가 가격은 `calc_limit_up_price()` 또는 REST `stck_mxpr` 필드 활용

### 4-2. 수량 계산

```python
# Signal Strength에 따른 수량 조절
max_invest = INIT_CASH * 0.1   # 1종목 최대 투자금 = 초기자금의 10%
base_qty = int(max_invest / buy_price)

if strength >= 0.7:
    qty = base_qty              # 정상 수량
elif strength >= 0.5:
    qty = max(1, base_qty // 3) # 1/3 수량
else:
    qty = 0                     # 매수 안 함
```

### 4-3. 기존 29% 긴급매도와의 관계

**현재**: 보유 종목이 29%+ 도달 시 → 긴급매도 (`EMERGENCY_29PCT_SELL`)
**신규**: 미보유 종목이 25~28% 도달 시 → 매수 검토

**충돌 방지**:
- 이 전략으로 매수한 종목은 `EMERGENCY_29PCT_SELL` 대상에서 **제외**해야 함
- 매수 목적이 상한가 도달이므로, 29% 도달 시 매도하면 전략 목적에 반함
- 대신 별도의 손절 기준 적용 (아래 5번 참조)

## 5. 매도 (Exit) 전략

### 5-1. 상한가 도달 시

```
상한가 도달 (prdy_ctrt ≥ 29.5% 이상, 상한가 근접)
  → 즉시 매도하지 않음
  → 익일 전략으로 전환:
    - 익일 갭 상승 시 → 시초가 매도 (갭 이익 확정)
    - 익일 상한가 유지 시 → 보유 지속
    - 익일 하락 시 → 손절 기준에 따라 매도
```

### 5-2. 상한가 미도달 시 — 손절

```
[타임아웃 손절]
매수 후 10분 경과 + 등락률 25% 미만으로 하락 → 즉시 매도

[가격 손절]
매수가 대비 -3% 하락 → 즉시 매도

[트레일링 스탑]
최고가 대비 -2% 하락 → 매도 (상한가 근접 종목은 변동폭이 크므로 타이트하게)
```

### 5-3. 시간대별 처리

```
14:50 이후 신규 매수 금지 (상한가 도달 시간 부족)
15:10 이후 미도달 종목 → 시장가 청산
15:20 이후 → 종가 매수 전략으로 자연 전환 (이미 보유 중이면 유지)
```

## 6. 보조 API 활용 상세

### 6-1. Volume Power (체결강도) — 매수 판단의 핵심

```
[호출 시점] 25~28% 구간 진입 감지 시 (1회)
[API] FHPST01680000 — /uapi/domestic-stock/v1/ranking/volume-power
[판단]
  - 해당 종목이 체결강도 상위 20% → +0.20 가산
  - 체결강도 하위 30% → -0.15 감산
[의미] 매수세가 매도세를 압도하는지 = 상한가까지 밀어올릴 힘이 있는지
```

### 6-2. 호가 잔량 분석 (WSS H0STASP0) — 매도벽 확인

```
[데이터 소스] WSS 실시간 호가 (이미 구독 중)
[판단]
  - 상한가 근처 매도 잔량 확인
  - 매도잔량이 매수잔량의 3배 이상 → 매도벽 = 상한가 도달 어려움 (-0.20)
  - 매수잔량이 매도잔량의 2배 이상 → 매수세 우세 (+0.15)
[핵심] 28% 가격에 대량 매도 주문이 쌓여있으면 → 상한가 도달 불가
```

### 6-3. 외국인/기관 동향 (frgnmem_pchs_trend) — 스마트머니 필터

```
[호출 시점] 장 시작 전 감시 종목에 대해 일괄 조회 (캐싱)
[API] FHKST644400C0 — /uapi/domestic-stock/v1/quotations/frgnmem-pchs-trend
[판단]
  - 최근 3일 외국인 순매수 → +0.10 (기관 매집 = 추세 신뢰)
  - 최근 3일 외국인 순매도 → -0.10 (개인만 매수 = 주의)
[한계] 일별 데이터이므로 당일 장중 변화 반영 불가
```

### 6-4. 상하한가 포착 (capture_uplowprice) — 사전 감시

```
[호출 시점] 매 3분 (09:10 ~ 14:50)
[API] FHKST130000C0 — /uapi/domestic-stock/v1/quotations/capture-uplowprice
[파라미터]
  FID_PRC_CLS_CODE = "0"   (상한가)
  FID_DIV_CLS_CODE = "3"   (25% 근접)
[활용]
  - WSS 미구독 종목 중 25% 근접 종목 발견 → 동적 구독 추가
  - 이미 구독 중이면 → 매수 판단 로직 즉시 실행
```

### 6-5. 변동성 확인 — 돌파 품질 판단

```
[계산 시점] 장 시작 전 1회 (일봉 기반)
[데이터] kis_1d_Daily_ohlcv_fetch_manager.py에서 수집한 일봉 OHLCV
[계산] 10일간 일간수익률의 표준편차
[판단]
  - vol < 2% (저변동성 → 돌파) → +0.10 가산
  - vol > 5% (고변동성 → 잡음) → -0.10 감산
[의미] 평소 조용하던 종목이 갑자기 25%+ 상승 = 강한 재료 시사
```

### 6-6. VI 해제 후 반응 — 가장 강력한 시그널

```
[감지] WSS 실시간 (추가 API 호출 불필요)
[조건] VI 해제 후 1분 이내 가격이 VI 발동가 이상으로 회복
[판단] +0.10 가산
[의미]
  - VI 해제 후 즉시 재상승 = 매수세가 VI로 꺾이지 않음 = 상한가 도달 가능성 높음
  - VI 해제 후 하락 지속 = 매수세 소진 = 보류
```

## 7. 리스크 관리

### 7-1. 상한가 근접 매수의 고유 리스크

| 리스크 | 확률 | 대응 |
|--------|------|------|
| **상한가 실패 → 급락** | 높음 (40~50%) | 타이트한 손절 (-3%), 10분 타임아웃 |
| **익일 갭 하락** | 중간 (30~40%) | 익일 시초가 -5% 이하 시 즉시 매도 |
| **작전주 / 세력 이탈** | 낮음 (5~10%) | 관리종목 필터, 거래량 최소 기준 |
| **상한가 도달 후 풀림** | 중간 (20~30%) | 상한가 잔량 모니터링 |

### 7-2. 포지션 사이징

```
1종목 최대 투자: INIT_CASH × 10%
동시 보유 상한: 최대 2종목 (상한가 근접 매수 전략)
일일 최대 매수: 3회 (실패 시 추가 진입 제한)
```

### 7-3. 시간대 제한

```
09:30 이전: 매수 금지 (장 초반 변동성 과다)
09:30 ~ 14:50: 매수 허용
14:50 이후: 신규 매수 금지
15:10 이후: 미도달 종목 청산
```

- 백테스트 최적 조합 평균수익 **-1.20%** (한 건의 -31% 대폭락 영향) → 구현은 **단계적 청산 + -5% 대폭락 방지** 로 완화 시도. 실전 50건 이상 누적 후 평균수익 재평가 필요.
- 백테스트에 없던 실시간 필터(체결강도, 호가잔량, 외국인) 추가 → 목표 승률 ≥ 65%, 상한가 도달률 ≥ 80%
- frgnmem_pchs_trend TR 검증 실패 시 F13 필터는 항상 통과 → 백테스트 조건과 괴리 가능
