### VI 매수 조건 — 매우 단순
```
antc_prce > stck_prpr  (예상체결가 > 현재가)
```

### 1. Volume Power API (체결강도 상위)

#### API 정보
| 항목 | 내용 |
|------|------|
| TR_ID | `FHPST01680000` |
| URL | `/uapi/domestic-stock/v1/ranking/volume-power` |
| 화면번호 | 20168 |

#### 주요 파라미터
```
fid_cond_mrkt_div_code = "J"     # KRX
fid_cond_scr_div_code = "20168"  # 고정값
fid_input_iscd = "0000"          # 전체 (또는 "0001":코스피, "1001":코스닥)
fid_input_price_1 = ""           # 가격 하한
fid_input_price_2 = ""           # 가격 상한
fid_vol_cnt = ""                 # 거래량 하한
```

#### 적용 방안: VI 매수 전 거래량 강도 필터

**현재**: VI 발동 → 예상체결가 > 현재가 → 발동 중 매수 주문 → VI 해제 시 체결
**개선**: VI 발동 → **체결강도 확인** → 예상체결가 > 현재가 → 발동 중 매수 주문 → VI 해제 시 체결

※ 매수 주문은 VI 발동 중(해제 전)에 넣으므로, REST API 호출 지연(0.5~1초)은 문제되지 않음 (VI 발동 지속시간 약 2분)

```
[적용 로직]
1. VI 발동 감지 시, 해당 종목의 체결강도 조회 (발동 중 2분 내 충분)
2. 체결강도가 일정 수준 이상인 경우에만 매수 진행
   - 강한 체결강도 (상위권): 정상 수량 매수
   - 약한 체결강도 (하위권): 매수 보류 또는 수량 축소
```

#### 기대 효과
- **거래량 뒷받침 없는 허수 VI 필터링** — VI 발동은 됐지만 실제 매수세가 없는 종목 제거
- **승률 향상** — 강한 수급이 확인된 VI만 진입

### 2. 예상체결가 호가 API (inquire_asking_price_exp_ccn)

#### API 정보
| 항목 | 내용 |
|------|------|
| TR_ID | `FHKST01010200` |
| URL | `/uapi/domestic-stock/v1/quotations/inquire-asking-price-exp-ccn` |
| HTS 화면 | 주식현재가 호가/예상체결 |

#### 주요 파라미터
```
FID_COND_MRKT_DIV_CODE = "J"       # KRX
FID_INPUT_ISCD = "005930"          # 종목코드 6자리
```

#### 응답 구조
- `output1`: 매수/매도 10호가 (호가잔량, 호가가격)
- `output2`: 예상체결 정보 (예상체결가, 예상체결량)

#### 적용 방안: VI 해제 후 회복 가격 예측

```
[적용 로직]
1. VI 발동 중 호가 조회 → 매수/매도 잔량 비율 분석
2. 매수잔량 >> 매도잔량: 해제 후 상승 가능성 높음 → 매수 강도 UP
3. 매도잔량 >> 매수잔량: 해제 후 하락 가능성 → 매수 보류
4. 호가 스프레드가 넓으면 (유동성 부족) → 매수 보류
```

#### 기대 효과
- **호가 잔량 분석으로 VI 해제 후 방향 예측** — 현재는 예상체결가 하나만 보고 판단
- **유동성 리스크 감소** — 호가가 비어있는 종목에 진입하는 실수 방지
- **슬리피지 예측** — 시장가 주문 시 실제 체결 예상가를 사전 파악
- **주의**: 현재 WSS로 H0STASP0(호가) 구독 중이므로, REST 호출 대신 WSS 데이터 활용이 더 빠름

### 3. Volatility Expansion 전략 로직 (변동성 확장)

#### KIS 공식 예제 전략 (strategy_08_volatility.py)

```
[매수 조건] (AND)
├─ N일(기본 10일) 변동성 < 0.02 (변동성 2% 미만 = 압축 상태)
└─ 당일 등락률 >= 3% (돌파 발생)

[매도 조건] (OR)
└─ 당일 등락률 <= -3% (하락 반전)

[손절] 3%
```

#### 핵심 개념: "변동성 압축 → 돌파"

```
변동성 압축 구간 (조용한 기간)
  ──────────────────── → 갑자기 3%+ 급등 → VI 발동!
                                              ↑
                                         이 시점에서 매수
```

- 변동성이 **낮았던** 종목이 갑자기 급등하면 → 추세 전환 신호 (강한 매수)
- 변동성이 **이미 높았던** 종목이 VI 발동하면 → 단순 변동 (약한 매수)

#### 적용 방안: VI 매수 품질 필터

```
[적용 로직]
1. 매일 장 시작 전, 감시 종목의 N일 변동성 계산
   - 변동성 = 일간 수익률의 표준편차 (10일 기준)
2. VI 발동 시, 해당 종목의 사전 변동성 확인:
   - 저변동성 종목 (< 2%) → 돌파 신호 강함 → 정상 매수
   - 고변동성 종목 (> 5%) → 일상적 변동 → 수량 축소 또는 보류
```

#### 기대 효과
- **VI 매수 신호의 품질 차별화** — 모든 VI를 동일하게 취급하지 않음
- **"진짜 돌파"와 "잡음" 구분** — 이미 널뛰기하는 종목의 VI는 의미가 약함
- **구현 난이도 낮음** — 일봉 OHLCV 데이터는 이미 수집 중 (kis_1d_Daily_ohlcv_fetch_manager.py)
- **주의**: 장중 실시간 계산이 아닌, 전일까지의 일봉 기반이므로 장 시작 전 1회 계산이면 충분

### 4. Signal Strength 개념 (신호 강도 체계)

#### KIS 공식 Signal 구조

| 강도 범위 | 분류 | 주문 방식 |
|-----------|------|----------|
| 0.8 ~ 1.0 | 강한 시그널 | 시장가 주문 |
| 0.5 ~ 0.8 | 일반 시그널 | 지정가 주문 |
| 0.0 ~ 0.5 | 약한 시그널 | 주문 보류 |

#### 적용 방안: VI 매수 강도 점수화

현재는 "매수한다 / 안 한다"의 이분법 → **0~1 점수로 변환하여 수량 조절**

```
[VI 매수 강도 산출 예시]

기본 점수: 0.5 (VI 발동 + 예상체결가 > 현재가)

가산 항목:
  +0.15  체결강도 상위 30% 이내
  +0.10  호가 매수잔량 > 매도잔량 × 1.5
  +0.10  저변동성 종목 (10일 vol < 2%)
  +0.10  외국인 순매수 중
  +0.05  거래량 전일 대비 200% 이상

감산 항목:
  -0.15  호가 매도잔량 > 매수잔량 × 2.0
  -0.10  고변동성 종목 (10일 vol > 5%)
  -0.10  외국인 순매도 중
  -0.10  호가 스프레드 2% 이상

최종 점수에 따른 행동:
  0.8+  → 최대 수량 시장가 매수 (강한 확신)
  0.5~0.8 → 기본 수량 매수
  0.3~0.5 → 최소 수량 매수 (테스트)
  0.3 미만 → 매수 보류
```

#### 기대 효과
- **수량 조절로 리스크 관리** — 확신 높을수록 많이, 불확실하면 적게
- **다중 필터의 체계적 통합** — 위 1~3번 API를 하나의 점수로 합산
- **백테스트 및 튜닝 가능** — 각 항목의 가중치를 조절하며 최적화
- **구현 방식**: `vi_buy_strategy()` 함수에서 bool 대신 float 반환 → 수량 계산에 반영

### 5. 외국인/기관 매수 동향 (frgnmem_pchs_trend)

#### API 정보
| 항목 | 내용 |
|------|------|
| TR_ID | `FHKST644400C0` |
| URL | `/uapi/domestic-stock/v1/quotations/frgnmem-pchs-trend` |
| HTS 화면 | [0433] 종목별 외국계 순매수추이 |

#### 주요 파라미터
```
FID_COND_MRKT_DIV_CODE = "J"
FID_INPUT_ISCD = "005930"     # 종목코드
FID_INPUT_ISCD_2 = "99999"    # 고정값
```

#### 적용 방안: 스마트머니 필터

```
[적용 로직]
1. VI 발동 종목에 대해 외국인/기관 매수 동향 조회
2. 최근 N일간 외국인 순매수 지속 → 신호 강도 가산 (+0.10)
3. 최근 N일간 외국인 순매도 지속 → 신호 강도 감산 (-0.10)

[활용 시점]
- 장 시작 전 감시 종목 리스트에 대해 일괄 조회 (캐싱)
- VI 발동 시 실시간 조회는 지연 우려 → 사전 캐시 활용
```

#### 기대 효과
- **외국인/기관이 매집 중인 종목의 VI = 추세 확인** → 매수 확신 강화
- **개인만 매수하는 종목의 VI = 주의** → 수량 축소
- **주의**: 일별 데이터이므로 장중 변화 반영 불가. 전날까지의 추세만 참고

#### 한계
- REST API 호출 시간 (0.5~1초) × 감시 종목 수 = 상당한 지연
- **장 시작 전 일괄 조회 후 메모리 캐싱**이 현실적
- 실시간 외국인 매매는 별도 API (`investor_trade_by_stock_daily`) 필요

### 6. 상하한가 포착 (capture_uplowprice)

#### API 정보
| 항목 | 내용 |
|------|------|
| TR_ID | `FHKST130000C0` |
| URL | `/uapi/domestic-stock/v1/quotations/capture-uplowprice` |
| HTS 화면 | [0917] 실시간 상하한가 포착 |

#### 주요 파라미터
```
FID_COND_MRKT_DIV_CODE = "J"
FID_COND_SCR_DIV_CODE = "11300"
FID_PRC_CLS_CODE = "0"          # 0:상한가, 1:하한가
FID_DIV_CLS_CODE = "1"          # 0:상하한가, 6:8%근접, 5:10%, 1:15%, 2:20%, 3:25%
FID_INPUT_ISCD = "0000"         # 전체
```

#### 적용 방안: VI 발동 전 사전 감지

```
[적용 로직]
1. 장중 주기적으로 (5분 간격) 15%~25% 상한가 근접 종목 조회
2. 근접 종목 리스트를 WSS 감시 대상에 동적 추가
3. 해당 종목이 VI 발동 시 → 이미 모니터링 중 → 빠른 대응

[또 다른 활용]
- 상한가 도달 종목 = VI 발동 가능성 높은 종목
- 미리 호가/체결강도를 캐싱해두면 VI 발동 시 즉시 판단 가능
```

#### 기대 효과
- **VI 발동 전 선제적 감시** — 현재는 VI 발동 후에야 종목을 인지
- **WSS 구독 효율화** — 유망 종목에 미리 예상체결가 구독
- **매수 반응 시간 단축** — VI 발동 시 이미 데이터가 준비되어 있음

#### 한계
- 40 슬롯 제한 → 감시 종목 무한정 추가 불가
- 5분 간격 조회 시 API 호출 횟수 관리 필요
- **현재 Top30 선정 로직과 통합** 필요 (중복 관리)
