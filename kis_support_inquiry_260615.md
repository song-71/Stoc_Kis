# 실시간 예상체결(H0STANC0) 데이터 깨짐 문의

**작성일:** 2026-06-15 / **대상:** Open API(WSS) 담당
**환경:** 국내주식 실시간시세 WebSocket, `H0STANC0`(실시간예상체결) 구독

## 1. 요지
실시간 예상체결(`H0STANC0`) 일부 레코드의 **필드 정렬이 깨져** 수신됩니다.
종목코드 칸이 비거나, 숫자 칸에 **공백**이 섞이고, 여러 필드가 **한 칸에 병합**되어
예상체결가가 수십~수백 자리 숫자로 들어옵니다. 같은 프레임에 **UTF-8에 없는
바이트(0xA3 0xAC, 0xEB 0xA3 0xAC 등)** 도 섞입니다.
**정규장 체결(`H0STCNT0`)은 동일 환경 41만 건 중 0건** 깨짐 — 예상체결에만 발생합니다.

## 2. 정상 vs 오류 (2026-06-15, 동일 08:50:02 구간)
문서상 `H0STANC0`는 45개 필드(`^` 구분), 다건은 `^`로 연결 → 45칸 단위로 분리합니다.

**정상 — 파크시스템스(140860):**
```
140860^085002^298500^5^-41000^-12.08^0.00^0^0^0^299000^293000^0^27^8059500^0^0^0^0.00^0^0^5^0.01^0.00^000000^3^0^…^20260615^00^N^11^1^16^6^0.00^0^0.00^B^
```
필드 45개, 종목코드 `140860`, 예상체결가 `298,500`, 등락률 `-12.08%` 정상.

**오류 — 같은 구간:**
```
^0   08^892350514^^-28561358^0.00^0.00^32^32^32^32^16691248^71578413126647808^587276648185856^…^137438953472^…
```
- 첫 필드(종목코드) **비어 있음**, 둘째 필드에 **공백**(`0   08`)
- 가격 칸 `892350514`(9자리), 뒤 칸들에 `71578413126647808`(17자리),
  `137438953472`(=2³⁷) 등 **여러 필드가 구분자 없이 병합**
- 더 심한 경우 한 필드에 **약 280자리 숫자**가 들어옴(다수 필드 병합).

## 3. 바이트 증거 (TLS 수신 = 서버 송신 바이트)
```
[08:50:29] H0STANC0 offset 353: …"-0.00^"  [A3 AC]  "1293.10"…
[08:50:33] H0STANC0 offset 358: …"-0.00^"  [EB A3 AC] …
```
ASCII 숫자/구분자 자리에 비정상 바이트가 삽입됩니다. 08:50~09:00 반복 발생.
