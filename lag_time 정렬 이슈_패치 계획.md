## 4. 정렬 키 검토 — acml_vol 가 견고한 tie-breaker

### 4.1 검증 결과 (인바이오젠 09:02:3X 86건)
- **recv_ts 순서로 본 acml_vol 의 역전 횟수: 0건**
- **acml_vol 정렬 후 stck_cntg_hour 단조증가: True**

→ KIS WSS 는 거래소 체결 순서를 그대로 패킷에 담아 보내며, 같은 도착 burst 안에서도 `acml_vol` 은 단조증가.

### 4.2 의미
- 실시간 도착 순서(`recv_ts`) 와 거래소 체결 순서(`acml_vol`) 는 **일치**한다.
- 즉 KIS WSS 의 도착 순서를 신뢰해도 거래소 진실 순서는 보장됨.
- `acml_vol` 의 가치는 다음 두 경우의 **tie-breaker / 견고성**:
  1. 같은 `recv_ts` ms 안에 다수 틱이 묶일 때 — 안정 정렬
  2. 데이터를 다시 저장/병합/리플레이 할 때 — `recv_ts` 가 손상되거나 ms 정밀도가 떨어져도 `acml_vol` 만으로 거래소 순서 복원 가능

→ 권장 정렬 키: `(mksc_shrn_iscd, stck_cntg_hour, acml_vol)` 또는 `(mksc_shrn_iscd, acml_vol)`.

## 5. 시뮬레이션·백테스트 영향

### 5.1 위험
- 백테스트가 단순히 `recv_ts` 정렬로 진행하면, 같은 거래소 초의 burst 들이 1~3초 늦게 묶여 들어오는 데이터 패턴 그대로 시뮬레이션됨.
- 결과: "이 가격에 매수/매도 가능했다" 라는 시뮬레이션 판정이 실제 거래소 시각이 아닌 도착 시각 기준이 됨.
- 트레일·매수 트리거 시각이 실제보다 늦게 평가되어 시그널/슬리피지 평가가 왜곡될 가능성.

### 5.2 점검 대상
- `symulation/Query_str_*_simulation.py` 가 어느 컬럼으로 정렬·진행하는지
- 1m / 1d aggregation 시 sub-second 순서가 의미 있는지

### 5.3 권장 변경
- 정렬 키를 **`(mksc_shrn_iscd, acml_vol)`** 또는 **`(mksc_shrn_iscd, stck_cntg_hour, acml_vol)`** 로 통일
- 시뮬레이션이 본 가격 시점 = 거래소 시점에 일치
- "lag 만큼 늦게 본 가격으로 매매" 라는 보수적 시뮬레이션이 필요하면 별도 옵션으로 분리

## 6. 실시간 매매 관점 — 시장가 정당성 vs lag 영향

### 6.1 시장가 매수/매도 결정의 트레이딩 논리
사용자는 의도적으로 시장가를 선택. 근거:
- **횡보 구간**: 지정가도 잘 체결됨
- **등락 구간**: 지정가는 가격이 빠르게 움직여 미체결 위험 큼 → 슬리피지 감수하고 시장가
- 모멘텀 매매/추세 진입·정리에는 미체결 회피가 슬리피지보다 더 중요

→ **트레이딩 측면에서 정당**. 인바이오젠 케이스도 매수1호가가 9,250 → 9,200 → 9,210 으로 격변 중이었고, 지정가 9,250 으로 걸었으면 미체결 후 더 빠진 호가에 노출될 가능성 컸음.

### 6.2 lag 가 시장가에 미치는 진짜 위험
- 시장가 매도 자체는 슬리피지 감수 정책으로 정당.
- 그러나 **lag 가 큰 burst 에서 트레일 트리거가 발동하는 것은 별개 문제**.
- 사례: trail_level 9,253.8 < bidp1 9,250 으로 트리거됐는데, 그 bidp1 정보가 1~3초 전 거래소 상태였다면 — 실제 시장은 이미 9,210까지 빠진 상태에서 우리만 "지금 매도 시작" 결정.
- 즉 슬리피지 = (lag 동안 시장이 빠진 폭) + (시장가 매칭 슬리피지) 의 합.

### 6.3 시장가 유지 + lag 가드 — 최적 절충
- 시장가 매도/매수 정책은 그대로 유지 (등락구간 미체결 회피)
- 단 **lag > 1.0초 burst 에서는 트레일 트리거 자체를 보류** — 잘못된 호가 정보 기반 판단 회피
- 정상 lag burst 진입 후 트리거 → 시장가 매도 → 슬리피지 최소화

→ 시장가 vs 지정가 논쟁이 아닌, **lag 가드로 잘못된 트리거 자체를 막는 것**이 본질.

## 9. 부록 — 측정 재현

```python
# 09:02:30~35 인바이오젠 추적 (방금 분석에 사용)
files = sorted(glob.glob('data/wss_data/parts/260529_*.parquet'))
df = pl.concat([pl.read_parquet(f) for f in files], how='diagonal_relaxed')
df = df.filter(pl.col('tr_id') == 'H0STCNT0')

# lag 계산
df = df.with_columns([
    pl.col('recv_ts').str.strptime(pl.Datetime('us'), format='%Y-%m-%d %H:%M:%S%.f').alias('recv_dt'),
    pl.col('stck_cntg_hour').str.slice(0,2).cast(pl.Int64).alias('eh'),
    pl.col('stck_cntg_hour').str.slice(2,2).cast(pl.Int64).alias('em'),
    pl.col('stck_cntg_hour').str.slice(4,2).cast(pl.Int64).alias('es'),
])
df = df.with_columns([
    (pl.col('eh')*3600 + pl.col('em')*60 + pl.col('es')).alias('exch_sec'),
    (pl.col('recv_dt').dt.hour().cast(pl.Int64)*3600
     + pl.col('recv_dt').dt.minute().cast(pl.Int64)*60
     + pl.col('recv_dt').dt.second().cast(pl.Int64)
     + pl.col('recv_dt').dt.nanosecond().cast(pl.Int64)/1e9).alias('recv_sec'),
])
df = df.with_columns([
    (pl.col('recv_sec') - pl.col('exch_sec')).alias('lag_raw'),
    (pl.col('recv_sec') - pl.col('exch_sec') - 0.5).alias('lag_est'),  # ms 평균 보정
])
```
