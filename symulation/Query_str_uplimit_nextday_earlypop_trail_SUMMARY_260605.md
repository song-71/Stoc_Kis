| 항목 | 내용 |
|------|------|
| **유니버스** | 전일(D-1) 상한가 종목 (CSV의 `date`=상한가일, 매매는 다음 거래일 D) |
| **진입 필터** | D일 09:00 **시초 갭 0% ≤ gap < 3%** (gap = D시가/전일종가−1). 적당한 갭상승만. |

| **청산** | 진입 후 **종가 고점(peak) 대비 −1~2% 트레일링 스톱** (low ≤ peak×(1−TRAIL) → 청산). 익절캡 없음. 미발생 시 마감 청산. |

## 💡 왜 작동하는가 (핵심 메커니즘 — 데이터로 검증)

1. **초반 고점 후 소멸**: 전일 상한가 종목 익일은 09:00 매수 시 **평균 고점이 09:02~09:05(+5.3%)**, 이후 흘러내려 **마감 +1.9%로 소멸**.
   - 분별 청산 평균: 09:01 −0.0% / **09:02 +5.3% / 09:05 +5.2%** / 09:10 +4.8% / 09:30 +4.3% / 마감 +1.9%.
   - → 초반 급등을 트레일링으로 잡고 반전 즉시 청산하는 게 정답.
2. **갭 0~3%에 수익 집중**: 트레일 −1% 기준 갭버킷별 평균 — 갭다운 +0.5%(승28%) / **갭0~3% +10.6%(승67%)** / 갭3~7% +0.4%(승20%) / 갭≥7% +0.6%(승29%).
   - 갭다운=약세, 갭≥3%=과열(gap & crap) → 둘 다 트레일링해도 못 범. **적당한 갭상승만 건강한 지속.**
3. **익절캡 금지**: 수익분포가 우측 팻테일(소수가 +30%까지). RSI>70·고정% 익절캡을 씌우면 그 꼬리를 잘라 **승률↑인데 총수익은 마이너스**가 됨(−7%손절+3%캡 → 승률66.7%인데 −0.08%).

| 강한 초반 종목 09:30 진입 | −1.8% | **룩어헤드 제거 시 소멸** (09:30엔 이미 다 올라 페이드 시작) |

## ⚠️ 한계 & 실전 적용 전 필수 검증

1. **단일 표본(2026.3~6월, 57일, 단일 국면)** — **아웃오브샘플(예: 1~2월) 재검증 필수.** 과적합/국면의존 점검.
2. **체결 슬리피지** — 백테스트는 트레일선에 정확히 체결 가정. 빠른 1분봉 급락 시 −1% 타이트 트레일은 더 나쁘게 체결될 수 있음 → **−2~3% 보수화 권장**.
3. **자동매매 필수** — 09:02 고점·분 단위 트레일은 수동 불가(하루 ~8종목 동시). `ws_realtime_trading.py`에 실시간 트레일링 청산 로직 구현 필요.
4. **갭 0~3% = 사후 버킷 선택** — 경제적 논리는 명확하나(과열/약세 제외) OOS로 안정성 확인 권장.
5. 종목당 비중·동시보유·자금곡선/MDD는 미반영(평균/총합 기준).
